1. Introduction

The MESC-Mathematica IDE is an advanced, cell-based computational notebook interface integrated into the broader MESC ecosystem. Designed to closely emulate the user experience and syntax of Wolfram Mathematica, this environment provides researchers, scientists, and students with a robust platform for symbolic mathematics, algorithmic execution, and interactive data visualization.

1.1 Core Objectives

  • Provide a seamless, interactive notebook environment for step-by-step code execution.
  • Render complex mathematical outputs natively using high-quality typography.
  • Ensure data resilience through robust manual and automatic session persistence mechanisms.
  • Offer accessible floating tool palettes for rapid mathematical input.

2. Technologies Used

The Mathematica IDE relies on a hybrid architecture, combining the performance of a native desktop framework with the flexibility of modern web rendering engines.

  • Python 3.11+: The core execution language handling process routing.
  • SymPy: The backbone for computer algebra and symbolic mathematics parsing.
  • PySide6 (Qt for Python): Utilized for the native application window, system dialogs, and native toolbars.
  • QtWebEngine: An embedded Chromium-based browser engine used to render the highly interactive notebook interface.
  • MathJax 3.0: Asynchronously renders LaTeX and MathML outputs into crisp, scalable mathematical notation.

3. Architecture Overview

The IDE operates on a bridge-communication model between the native Qt environment and the embedded JavaScript DOM:

  1. Input: The user types math into a JS <textarea>.
  2. Trigger: Pressing Shift + Enter triggers a JS event.
  3. Bridge: JS sends a base64 encoded string via console.log("EXECUTE:...").
  4. Intercept: PySide6's QWebEnginePage.javaScriptConsoleMessage intercepts the log, decodes the string, and passes it to the Python backend.
  5. Computation: Python evaluates the math using SymPy.
  6. Output: Python generates HTML/LaTeX, encodes it in base64, and injects it back into the DOM.
  7. Render: MathJax processes the newly injected DOM element.

4. User Guide

4.1 The Notebook Interface

The primary workspace is the Notebook, consisting of consecutive input and output cells.

  • Execution: Type your mathematical expression and press Shift + Enter to evaluate it.
  • System Messages: Errors, kernel resets, and file-loading confirmations appear in distinct, blue-bordered system cells.

4.2 Toolbar Controls

  • 📂 Load Notebook: Opens a .mathnotebook JSON file and restores the visual layout and backend variables.
  • 💾 Save Notebook: Serializes the current DOM state and Python variables into a standalone file.
  • 🗑️ Clear Notebook: Wipes the DOM and resets the cell counter visually.
  • 📄 Export to PDF: Renders the current notebook structure into a high-fidelity PDF document.
  • 📋 Variables: Opens the Variable Inspector Dialog, presenting a real-time table of all assigned symbolic variables.

4.3 Floating Palettes

Floating palettes can be triggered from the "Palettes" dropdown menu. These windows utilize Qt.Tool to remain permanently on top of the IDE without stealing keyboard focus from your active cell.

  • Basic Math Assistant: Access to common operators (√, x², π, e), Calculus templates (∫, ∂, ∑), and common functions (Sin, Log).
  • Special Characters: A comprehensive grid of Greek letters (α, β, Γ, Δ) and relational operators (≤, ≥, ≠, →).
Note: Clicking a palette button automatically inserts the requested symbol at your exact cursor position within the currently active input cell.

5. Advanced Capabilities

5.1 Zero-Loss Persistence (Continuous Auto-Save)

The IDE guarantees zero data loss in the event of unexpected app closure or system failure.

  • Every time a user changes focus or executes code, the system triggers autoSave().
  • The entire DOM structure and SymPy backend variable dictionary are serialized into JSON.
  • This JSON is piped into OS-native QSettings. Upon the next application launch, the exact DOM state and variables are rebuilt seamlessly.

5.2 Dynamic Variable Manipulation

The IDE supports dynamic variable tweaking. When a slider or interactive element is moved in the output cell, Python temporarily updates the global variable dictionary, re-evaluates the mathematical expression, and patches only the specific HTML container of that cell ID to prevent page stuttering.

6. Developer Guidelines

6.1 Adding New Palette Items

To add a new symbol to a palette, navigate to ides/mathematica/ui/palettes.py. Locate the relevant array and append a tuple: ("Button Label", "TextToInsert").

# Example: Adding a Matrix template
matrix_buttons = [
    ("2x2 Matrix", "{{a, b}, {c, d}}")
]
self.add_section("Linear Algebra", matrix_buttons, columns=1)

6.2 The JavaScript/Python API Contract

When altering the HTML template, adhere strictly to the base64-encoded console.log data bridge format:

  • Execution: console.log("EXECUTE:" + cell_id + ":" + base64_code);
  • Manipulation: console.log("MANIPULATE:" + cellId + ":" + varName + ":" + value + ":" + base64_expr);
  • State Saving: console.log("SAVE_STATE:" + base64_json_state);

7. Troubleshooting

Issue Cause Resolution
Math elements show raw LaTeX instead of formatting. MathJax CDN failed to load. Ensure active internet connection, or configure the IDE to use a locally downloaded MathJax library.
Clicking a Palette button deletes text instead of inserting. Focus tracking failed. The cursor activeCellId relies on the textarea onfocus event. Ensure you have clicked inside a cell at least once.
App lags severely during loops. maxCellId sync error. Do not manipulate DOM elements directly through python loops. Use the asynchronous appendSystemMessage method.
Variables lost after saving notebook. Un-serializable SymPy objects. If a custom object lacks a string representation, it cannot be recovered via sympy.sympify().
